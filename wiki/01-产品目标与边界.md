# 产品目标与边界

本页记录实现必须满足的产品契约。用户介绍见 `README.md`。

## 功能契约

- 支持 `morning` 和 `evening` 两种 edition。
- 晨报建立当日基线；晚报读取当日晨报、已有评估和人工反馈。
- 报告固定包含七个内容栏目：
  - `information.international`
  - `information.domestic`
  - `information.military`
  - `information.market`
  - `technology.news`
  - `technology.papers`
  - `technology.open_source`
- 研判固定覆盖 `geopolitics`、`ai_technology` 和 `markets`。
- 本地 JSON 和 Markdown 是事实源；HTML、PDF 和 Notion 是投影。
- Notion 未配置或发布失败时，本地交付仍可完成。

## 内容层级

| 层级 | 用途 | 约束 |
| --- | --- | --- |
| `ArticleItem` | 采集候选 | 由来源、规范 URL 和采集时间标识 |
| `brief` | 栏目覆盖 | 标题、中文 TL;DR、来源排序和可选精选事件引用 |
| featured event | 精选事实 | schema 1.5 中绑定一个来源条目 |
| `analysis` | 研判 | 只能引用报告内已有事件 |

普通候选不需要读取正文或生成研判。正文读取和深度分析只用于有限数量的精选事件。

## 证据约束

- 标题、URL、来源身份和访问状态必须来自 Index。
- 未读取正文时，摘要只能使用标题或公开元数据明确表达的信息，并标记证据边界。
- 访问失败不能转换为 `no_items`。
- 分析中的事实和具名来源必须能回溯到 `evidence_event_ids`。
- 用户可见文本使用中文；原文标题可以保留原语言。

## 时间与连续性

- 所有日期和 edition 窗口按配置时区计算。
- 晚报需要提供 `changes` 和 `tomorrow_watch_items`。
- 只有通过独立评估门槛的语义内容才能进入后续复用。
- 报告、Index 和 Evaluation 使用 Revision 或内容 Hash 绑定，不覆盖旧版本。

## 非目标

- 不绕过登录、验证码、限流、付费墙或站点访问控制。
- 不保存或发布 Cookie、浏览器 Profile、原始认证页面。
- 不在 Python 中生成自然语言摘要或研判。
- 不因重要性较低而静默删除来源覆盖目标内的条目。
- 不执行交易，也不提供个性化投资、法律或其他专业建议。

## 兼容要求

- 继续读取旧版 source-index 的根级和嵌套 `items` 结构。
- 继续读取 schema 1.1 至 1.4 报告。
- 兼容字段必须显式归一化；不允许静默改变语义。
- 删除兼容路径前，需要迁移说明和覆盖旧格式的测试。
