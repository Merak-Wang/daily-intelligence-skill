# 数据与状态模型

## 采集实体

`models.py` 定义两个主要数据类：

| 类型 | 关键字段 |
| --- | --- |
| `ArticleItem` | `item_id`、`source_id`、`title`、`url`、`canonical_url`、`module`、`category`、`content_status` |
| `SourceResult` | `source_id`、`source_url`、`status`、`http_status`、`challenge`、`page_results`、`items` |

`item_id` 由 `source_id` 和规范 URL 生成。相同来源和规范 URL 必须得到相同 ID。

## 来源状态

`SourceStatus`：

| 状态 | 含义 |
| --- | --- |
| `success` | 来源成功且有候选 |
| `partial` | 部分页面或条目可用 |
| `verification_required` | 需要合法人工验证 |
| `rate_limited` | 命中限流或临时限制 |
| `failed` | 请求或解析失败 |
| `no_items` | 页面可访问且确认没有合格条目 |

HTTP 401、403、429、挑战页和访问限制不能记为 `no_items`。

## 正文状态

`ContentStatus`：

| 状态 | 含义 |
| --- | --- |
| `not_fetched` | 尚未尝试读取正文 |
| `full_text` | 已保存可用正文 |
| `partial` | 只读取到部分正文 |
| `metadata_only` | 仅有标题或公开元数据 |
| `verification_required` | 正文页需要人工验证 |
| `failed` | 正文读取失败 |

报告中的 `access` 由该状态映射，不能由模型自行声明。

## Run 状态

`workflow.RunStatus`：

```text
created
  → collecting
  → building_context
  → awaiting_selection
  → extracting_content
  → awaiting_authoring
  → finalizing
  → publishing
  → completed | completed_partial

任一阶段可进入 failed；可恢复错误会返回相应 awaiting 状态。
```

Run manifest 保存当前状态、阶段时间、Artifact 路径、已选 ID、错误、发布和评估调度信息。

## 报告模型

报告契约位于 `schemas/report.schema.json`。当前写作流程使用 schema 1.5，同时保留 1.1 至 1.4 的读取兼容。

| 字段 | 说明 |
| --- | --- |
| `report_id`、`revision` | 报告身份 |
| `date`、`edition`、`generated_at` | 时间身份 |
| `sections[].briefs` | 来源覆盖层 |
| `sections[].items` | 精选事件层 |
| `analyses` | 三类研判 |
| `changes` | 晚报相对晨报的变化 |
| `tomorrow_watch_items` | 晚报次日观察 |
| `pending_verifications` | 未完成访问来源 |
| `source_metrics` | 配置、成功、代表和待处理计数 |

模型草稿不需要填写所有确定性字段。`compile_report_data()` 负责 ID、引用、计数、来源排序、状态和重要性拆分。

## Evaluation

Evaluation 单独保存，至少包含：

- 被评报告 ID 和内容 Hash；
- 九个固定评分维度；
- 总分、缺陷、证据不足和修改项；
- 连续性决定及排除范围。

Evaluation 不修改原报告。页面投影可在 Evaluation 完成后刷新质量区域。

## 文件布局

```text
DATA_DIR/
├─ runs/YYYY-MM-DD/{edition}.json
├─ indexes/YYYY-MM-DD/{edition}-rN.json
├─ context/YYYY-MM-DD/{edition}-rN.json
├─ content/{source_id}/{item_id}/...
├─ reports/YYYY-MM-DD/{edition}-rN.{json,md,html,pdf}
├─ evaluations/YYYY-MM-DD/...
├─ state/
├─ publishing/
├─ challenges/
└─ locks/
```

路径必须位于当前绑定的数据根内。`runtime.py` 会拒绝跨数据根的 Run、Index 或 Report。

## 兼容结构

旧 source-index 可能同时包含：

- 根级 `items`；
- `sources[].items` 嵌套条目。

读取逻辑需要兼容两种结构。正文更新后由 `synchronize_nested_items()` 保持两层内容一致。
