# 端到端流程

## CLI 启动

`daily_intelligence.cli.main()` 按以下顺序初始化：

1. 解析 CLI 参数；
2. 加载 Hermes Home 下的 `.env`，不覆盖已有进程环境变量；
3. 加载来源配置；
4. 解析并校验唯一数据根；
5. 分发子命令。

## Prepare

命令：

```powershell
daily-intel run-edition --edition morning
daily-intel run-edition --edition evening
```

`prepare_edition()` 执行：

1. 获取 `{date}-{edition}` 排他锁；
2. 创建或恢复 Run manifest；
3. 晚报同步已有人工反馈；
4. 并发预取公开索引页；
5. 对需要浏览器的来源使用 Adapter 或 Edge；
6. 保存不可变 Index；
7. 读取历史报告、Evaluation 和连续状态；
8. 生成 Context、`brief_plan` 和最多三个写作批次；
9. 将 Run 置为 `awaiting_selection`。

默认不会打开人工验证窗口。显式传入 `--open-verification` 才会在采集后启动队列；Cron 不应使用该参数。

## Hermes 写作与选择

Hermes 从 Context 完成：

- 原文标题的中文翻译；
- 证据范围内的中文 TL;DR；
- 精选正文 ID；
- 精选事件和三个分析域的草稿。

Python 不补写缺失翻译、摘要或研判。缺失内容会在编译或校验阶段返回可操作错误。

## Enrich

命令：

```powershell
daily-intel enrich-edition --run <run.json> --item-id <item_id>
```

`enrich_edition()`：

1. 校验 Run 和 Index 属于当前数据根；
2. 合并本次与此前已选 ID；
3. 按配置硬上限截断；
4. 并发读取正文，同域受单独并发限制；
5. 保存正文 Markdown；
6. 生成新 Index Revision；
7. 重建 Context；
8. 记录 Enrichment lineage；
9. 将 Run 置为 `awaiting_authoring`。

多次 Enrich 会累积已接受 ID，不重复声称被硬上限截断的条目已读取。

## Finalize

命令：

```powershell
daily-intel finalize-edition --run <run.json> --report <draft.json>
```

可选 Notion：

```powershell
daily-intel finalize-edition --run <run.json> --report <draft.json> --publish
```

`finalize_edition()`：

1. 校验 Enrichment lineage 和当前 Index；
2. 编译模型草稿，注入确定性字段；
3. 执行 JSON Schema 和跨字段语义校验；
4. 保存不可变 JSON 与 Markdown；
5. 生成配置要求的 HTML、PDF 和本地日报索引；
6. 更新连续状态和语义缓存待评记录；
7. 可选发布 Notion；
8. 将 Run 置为 `completed` 或 `completed_partial`；
9. 调度独立 Evaluation。

报告校验失败时不保存或发布，Run 返回 `awaiting_authoring` 并记录错误。

## Evaluation

Evaluation 在本地交付后独立执行。完成后调用：

```powershell
daily-intel finalize-evaluation --report <report.json> --evaluation <evaluation.json>
```

该阶段：

- 校验报告 ID 和 Hash；
- 保存不可变 Evaluation；
- 更新语义缓存和连续状态；
- 刷新 HTML/PDF 的质量区域；
- 如果原报告已发布到 Notion，可显式追加 Evaluation。

## 人工验证

```powershell
daily-intel verify-pending --index <index.json> --timeout-seconds 300
```

验证队列只处理失败、挑战或需要验证的链接。成功捕获的页面合并为新 Index Revision；其他失败链接继续保留。已完成报告对应的 Run 会重新打开为可追踪的新 Revision，不会覆盖原报告。

## 恢复规则

| 情况 | 处理 |
| --- | --- |
| 来源部分失败 | 保留失败状态，继续构建 partial 报告 |
| 限流 | 停止该来源的浏览器重试，等待后续恢复 |
| 正文失败 | 保留元数据和访问状态，不伪造正文 |
| 草稿校验失败 | 返回 `awaiting_authoring` |
| Notion 失败 | 本地报告保持完成，发布登记可重试 |
| Evaluation 调度失败 | 记录调度状态，不回滚报告 |
| 锁已存在 | 报告正在运行或上次异常退出；检查锁内容后处理 |
