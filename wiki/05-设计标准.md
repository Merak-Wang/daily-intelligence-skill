# 设计标准

## 实现约束

1. 确定性状态转换写在 Python 中，不写在模型提示词中。
2. 外部网页、API 响应和模型输出都按不可信数据处理。
3. 本地 JSON 和 Markdown 是事实源；远程发布必须可重试。
4. 访问失败、空结果、部分成功和限流使用不同状态。
5. 带 Revision 的 Artifact 不覆盖写入。
6. 可变 JSON 和文本使用原子写。
7. 并发共享资源使用显式锁和有界并发。
8. 路径在使用前解析，并校验位于当前数据根内。
9. 错误信息包含对象、阶段和修复动作。
10. 公开函数使用类型标注；状态使用显式枚举。

## 语义与确定性边界

Hermes 负责：

- 标题翻译；
- TL;DR；
- 精选事件选择；
- 分析、反证和观察信号。

Python 负责：

- ID、Revision 和 Hash；
- 来源、URL、访问状态和时间身份；
- 排序、数量限制和状态转换；
- Schema、跨字段校验和发布幂等；
- 文件写入和恢复信息。

Python 不使用模板句补齐缺失语义。Hermes 不生成权威来源身份或文件状态。

## 配置规则

- 固定行为进入类型化配置或枚举。
- 来源差异优先放入 `configs/sources.yaml` 或 Adapter。
- 动态来源页只允许配置域名范围内的 HTTP(S) URL。
- 默认值集中在 `config.py`，文档不复制易变化的完整默认配置。
- 环境变量和 CLI 参数冲突时必须报错或遵循已测试的明确优先级。

## 兼容规则

- 旧字段通过集中归一化读取，不在调用方散布兼容分支。
- 不修改历史不可变 Artifact。
- 兼容转换需要警告或迁移记录。
- 删除旧格式支持前，先提供迁移工具和测试。

## 测试要求

以下变更必须增加或更新测试：

- 来源 URL 或标题过滤；
- 来源、正文或 Run 状态；
- Schema、编译和跨字段校验；
- Artifact 路径、Revision、原子写或锁；
- HTML、PDF、Notion 或 Evaluation 发布；
- 旧 source-index 和旧报告兼容。

## 文档要求

- README 只描述用途、能力、安装和文档入口。
- Wiki 记录当前代码契约，不写路线图、宣传语或版本验收记录。
- 用户配置步骤放在 `references/`。
- 版本变化写入 `CHANGELOG.md`。
- 文档中的示例不得包含真实 Token、本机绝对路径或未经标记的合成新闻。
