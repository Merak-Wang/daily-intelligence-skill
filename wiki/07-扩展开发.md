# 扩展开发

## 添加普通来源

先在 `configs/sources.yaml` 添加来源。通用索引页不需要新增 Python 文件。

```yaml
- id: example_news
  name: Example News
  enabled: true
  mode: browser_index
  url: https://news.example/
  include_domains:
    - news.example
  article_patterns:
    - '^https://news\.example/articles/.+$'
  exclude_patterns:
    - '/authors/'
    - '/tags/'
  content_selectors:
    - article
    - main
  max_items: 60
  report_target: 10
  report_max: 15
  role: evidence
  module: information
  category: international
  language: en
  region: global
```

字段检查：

- `id` 稳定且唯一；
- `include_domains` 只包含允许域；
- `article_patterns` 只匹配文章；
- `exclude_patterns` 排除导航、作者、标签、登录和评论页；
- `module`、`category` 符合 `taxonomy.py`；
- `report_target <= report_max <= budget.report_items_per_source`。

至少增加一个接受文章 URL 和一个拒绝非文章 URL 的测试，位置通常是 `tests/test_normalize.py`。

## 添加多栏目页

固定栏目写入 `explore_urls`。运行时新增栏目使用：

```powershell
daily-intel source-page add `
  --source example_news `
  --url https://news.example/technology `
  --reason "新增技术栏目"
```

动态栏目保存在 `DATA_DIR/state/source-pages.json`，每个来源最多五个，且必须通过同域校验。不要把运行时栏目写回仓库配置。

## 添加 Adapter

仅在通用 `browser_index` 无法稳定表达来源协议时新增 Adapter，例如专用 API 或特殊 DOM。

1. 在 `adapters.py` 实现 `IndexAdapter`：

   ```python
   def collect_example(
       page: Page,
       source: SourceConfig,
       discovered_at: str,
   ) -> list[ArticleItem]:
       ...
   ```

2. 返回由 `_article()` 构造的 `ArticleItem`，不要自行生成不兼容 ID。
3. 将 Adapter 注册到 `_ADAPTERS`，或通过 `register_adapter()` 注册。
4. 在来源配置中设置 `mode` 或 `adapter`。
5. 为解析、去重、日期、上限、错误状态和异常响应增加测试。

Adapter 不负责写 Index、切换 Run 状态或发布报告。

## 添加分类

修改顺序：

1. `taxonomy.py` 的枚举、栏目 ID、顺序和标题；
2. `schemas/report.schema.json`；
3. `templates/report-contract.md`；
4. `reporting.py` 的归一化和校验；
5. Markdown、HTML、PDF 和 Notion 渲染；
6. Context 与来源配置；
7. 测试、Wiki 和 Changelog。

固定栏目变化属于报告契约变更，需要说明旧报告兼容策略。

## 添加报告字段

先确认字段归属：

- 模型语义字段：写入模板，由校验器检查；
- 确定性字段：由 `compile_report_data()` 注入；
- 发布字段：只存在于投影，不进入事实源；
- 运行字段：写入 Run 或 Index，不写入报告。

新字段需要覆盖：Schema、编译、校验、Markdown、本地 HTML/PDF、Notion、兼容读取和测试。

## 添加 Publisher

Publisher 只能接收已校验报告。实现时需要：

- 目标 schema 校验；
- 幂等键；
- 本地进度登记；
- 中断续传；
- 重发开关；
- 不修改事实源；
- 可操作的访问和字段错误；
- 单元测试使用 Fake Client，不依赖真实凭证。

## 变更检查

提交前确认：

- [ ] 新输入按不可信数据处理；
- [ ] 状态没有退化为布尔值或空列表；
- [ ] 路径位于活动数据根；
- [ ] 没有覆盖不可变 Artifact；
- [ ] 旧索引和旧报告仍可读取；
- [ ] 相关渲染器和发布器已同步；
- [ ] 新行为有回归测试；
- [ ] README 没有加入实现细节；
- [ ] Wiki 和 Changelog 已按需更新。
