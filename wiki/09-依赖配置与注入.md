# 依赖与配置

## Python 依赖

运行依赖定义在 `pyproject.toml`：

| 依赖 | 用途 |
| --- | --- |
| `httpx`、`beautifulsoup4` | 无脚本 HTTP 预取和 HTML 解析 |
| `playwright` | 浏览器采集、正文提取和验证 |
| `PyYAML` | 来源与发布配置 |
| `jsonschema` | 报告 Schema 校验 |
| `python-dotenv` | Hermes Home `.env` 加载 |
| `reportlab` | PDF 降级输出 |
| `tzdata` | 跨平台时区数据 |

开发依赖包含 pytest、Ruff 和 pypdf。

## 外部依赖

- Hermes Agent 与 Hermes Gateway：运行 Skill 和独立 Evaluation；
- Microsoft Edge：Windows 浏览器采集和默认 PDF 输出；
- Playwright Chromium：非 Windows 浏览器运行；
- Notion API：仅在显式发布时使用。

Python 包本身不调用模型供应商 API。

## 资源目录

`project_root()` 按顺序检查：

1. `DAILY_INTEL_SKILL_DIR`；
2. 当前 Python 包对应的源码根；
3. Hermes 标准安装目录；
4. 当前工作目录；
5. Hermes `skills/*/daily-intelligence`。

候选目录必须同时包含 `SKILL.md`、`configs/sources.yaml` 和 `schemas/report.schema.json`。

## 环境变量

| 变量 | 用途 |
| --- | --- |
| `HERMES_HOME` | 覆盖 Hermes Home |
| `DAILY_INTEL_SKILL_DIR` | 覆盖 Skill 资源目录 |
| `DAILY_INTEL_DATA_DIR` | 运行数据根 |
| `DAILY_INTEL_PROFILE_DIR` | 浏览器 Profile 目录 |
| `DAILY_INTEL_BROWSER_CHANNEL` | Playwright 浏览器 Channel |
| `NOTION_TOKEN` | 可选 Notion Token |
| `NOTION_DATA_SOURCE_ID` | 可选 Notion Data Source ID |

CLI 启动时加载 Hermes Home 下的 `.env`，`override=False`，因此不会覆盖进程已有环境变量。

## 参数优先级

### 来源配置

`--config` 指定 YAML；否则使用 Skill 内的 `configs/sources.yaml`。`--timezone` 覆盖 YAML 中的 `timezone`。

### 数据根

1. 显式 `--data-dir`；
2. `DAILY_INTEL_DATA_DIR`；
3. `<HERMES_HOME>/daily-intelligence`。

如果参数和环境变量同时存在且解析到不同目录，命令报错。只有 `data-root adopt` 允许处理该冲突。

### 浏览器 Profile

1. 子命令的 `--profile-dir`；
2. 配置指定名称的环境变量，默认 `DAILY_INTEL_PROFILE_DIR`；
3. `<HERMES_HOME>/browser-profiles/daily-intelligence`。

### 浏览器 Channel

1. 子命令的 `--browser-channel`；
2. `DAILY_INTEL_BROWSER_CHANNEL`；
3. YAML 的 `browser.default_channel`；
4. Windows 默认 `msedge`，其他平台为 Playwright 默认浏览器。

## 来源配置

`SourceConfig` 字段分为：

| 类别 | 字段 |
| --- | --- |
| 身份 | `id`、`name`、`enabled` |
| 入口 | `url`、`explore_urls`、`mode`、`adapter` |
| 过滤 | `include_domains`、`article_patterns`、`exclude_patterns`、`exclude_title_patterns` |
| 正文 | `content_selectors`、`wait_ms` |
| 数量 | `max_items`、`report_target`、`report_max` |
| 分类 | `role`、`module`、`category`、`language`、`region` |

加载时会校验 taxonomy 和报告数量关系。未知 Adapter 在采集时返回包含可用名称的错误。

## 浏览器与预算配置

`BrowserConfig` 控制 HTTP 预取、浏览器导航、正文提取的超时和并发。`BudgetConfig` 控制运行时限、Context 候选数、每来源报告上限和全文硬上限。

预算值必须通过 `AppConfig` 传入业务函数，避免模块内部复制数字常量。

## 输出配置

`OutputConfig` 支持 `html`、`pdf`：

- PDF 依赖 HTML；
- `pdf_engine` 可为 `edge`、`reportlab` 或 `auto`；
- Notion 不属于本地 `formats`，由 `--publish` 单独启用。

## 动态来源页

动态来源页保存在 `DATA_DIR/state/source-pages.json`，与仓库 YAML 分离。写入时校验：

- URL 是绝对 HTTP(S)；
- 主机属于来源 `include_domains` 或主入口域；
- 每个来源不超过五个动态页面；
- 状态为显式 `approved`。

## Notion 配置

`configs/notion.yaml` 定义 API 版本、属性名和固定选项。发布器读取目标 Data Source schema 后选择完全兼容的 Profile。未匹配时返回缺失或类型错误列表，不自动修改数据库。

Token 和 Data Source ID 的用户配置见 `references/notion-setup.md`。
