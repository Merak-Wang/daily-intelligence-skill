# 可靠性与安全

## 故障分类

| 故障 | 状态或处理 |
| --- | --- |
| 页面可访问但无合格链接 | `no_items` |
| 部分栏目页成功 | `partial` |
| 登录、403、挑战页 | `verification_required` |
| HTTP 429 或临时访问限制 | `rate_limited`，停止该轮浏览器重试 |
| 网络、解析或服务端错误 | `failed` |
| 正文不可访问 | 保留元数据，设置对应 `ContentStatus` |
| Notion 错误 | 保留本地报告，登记可重试状态 |

状态判断集中在 `access.py`、`collector.py`、`prefetch.py` 和 `content.py`。调用方不得自行把失败转换为成功或空结果。

## 写入与并发

- `write_json()` 和 `write_text_atomic()` 先写临时文件，再替换目标。
- `write_immutable_json()` 拒绝覆盖已有 Artifact。
- `next_revision()` 在目标目录中分配下一 Revision。
- `exclusive_lock()` 使用独占创建保护同一 edition。
- HTTP 预取和正文读取分别使用全局及同域信号量。
- 多进程共享可变状态时，不依赖内存锁。

锁文件包含 edition、时间和进程信息。遇到遗留锁时先确认对应进程和 Run manifest，不要直接批量删除锁目录。

## 数据根

`runtime.py` 负责：

- 绑定 Hermes 实例的唯一数据根；
- 拒绝 `--data-dir` 与环境变量冲突；
- 校验 Run、Index 和 Report 位于活动数据根；
- 防止不同手动任务、Cron 或安装副本混用历史。

迁移数据根必须通过 `daily-intel data-root adopt` 显式执行。

## 外部内容

网页、API 响应和模型草稿均为不可信输入：

- HTML 预取使用 BeautifulSoup 解析，不执行脚本；
- 浏览器页面只提取指定 DOM 数据，不执行页面中的文字指令；
- URL 必须通过协议、域名和正则过滤；
- 用户可见 HTML 对文本转义，并限制可点击 URL 为 HTTP(S)；
- PDF 生成不应加载任意远程资源；
- Notion 块只使用已校验报告字段。

网页内容不得修改来源配置、动态栏目、执行流程或本地文件。动态栏目只能通过 `source-page add` 显式批准，并受同域和数量限制。

## 浏览器数据

浏览器 Profile 只用于合法登录和人工验证。以下内容不得提交、复制到 Skill 安装目录或发布：

- Cookie 和 Storage State；
- 浏览器 Profile；
- HAR；
- 带账户信息的截图；
- 原始认证 HTML；
- 正文缓存和运行时 `data/`。

## 凭证

- Token 只通过环境变量或外部密钥管理器传入。
- `.env`、`.env.*` 被 Git 忽略，`.env.example` 只保留空键名。
- 错误消息不得输出 Token、Cookie 或完整认证响应。
- Token 泄漏后应立即轮换，并检查 Git 历史，而不仅是删除工作区文件。

## 发布

发布前必须通过报告校验和目标 schema 校验。发布器维护本地登记，记录页面 ID、已追加块数和状态。`--republish` 允许明确重发，但不跳过验证。

Notion、HTML 和 PDF 是可重建投影。任何投影失败都不能修改已保存报告的内容或 Hash。

## 安全检查

发布前至少执行：

```powershell
git status --short
git diff --check
rg -n -i "token|secret|cookie|storage.state|BEGIN PRIVATE" .
```

搜索结果需要人工区分占位符、代码中的变量名和真实凭证。
