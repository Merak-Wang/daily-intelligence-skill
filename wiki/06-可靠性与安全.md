# 可靠性与安全

## 1. 故障模型

系统把故障分成四层，恢复方式不同：

| 层 | 示例 | 处理 |
| --- | --- | --- |
| 来源层 | 403、429、CAPTCHA、DOM 变化 | 标记来源状态，继续其他来源 |
| 浏览器/本机层 | Edge 崩溃、profile 锁、超时 | 保留 run checkpoint，重试当前阶段 |
| 报告层 | Schema、中文、引用、覆盖率错误 | 回退 `awaiting_authoring`，修 draft |
| 发布/评估层 | Notion 429、Hermes Gateway 未运行 | 保留本地报告，稍后重试远端动作 |

“部分完成”是一等结果。只要报告本身合法，存在 pending source 时终态为 `completed_partial`，而不是整份失败。

## 2. 原子写与不可变写

`utils.write_json()` 和 `storage.write_text_atomic()` 都先写同目录临时文件，再 `replace()`，防止进程中断留下半个 JSON/Markdown。

`storage.write_immutable_json()` 在目标已存在时直接抛 `FileExistsError`。Index、Context、Report、Evaluation 和 state history 使用它；latest/current 视图可原子替换。

需要注意：写不可变 artifact 与更新 latest 不是一个跨文件事务。若进程恰好在两者之间退出，历史 revision 已存在且可靠，latest 可能滞后，可从最高 revision 重建。

## 3. 排他锁

每个日期和 edition 使用：

```text
locks/YYYY-MM-DD-<edition>.lock
```

锁通过文件的独占创建模式 `x` 获得，内容记录 PID、edition、created_at。正常退出在 `finally` 删除。

当前没有自动 stale-lock 判断。遇到残留锁时：

1. 检查 PID 是否仍在运行；
2. 检查 Hermes/Cron 是否仍在执行同一版；
3. 确认没有写文件后再手动删除。

不可为了“自动恢复”无条件删除锁，否则可能让手动任务和 Cron 同时写同一 run。

## 4. 浏览器隔离

使用专用持久化 profile：

- 保存必要登录状态，支持 Edge 人工验证；
- 避免与用户日常浏览器抢同一 profile lock；
- 降低无关扩展、标签和缓存影响；
- 便于整体删除和重新登录。

profile 目录属于敏感运行数据，不应进入仓库、日志附件或 bug report。

## 5. 网页内容不可信

外部页面可能包含提示注入、恶意 README、下载链接或诱导操作。系统边界是：

- adapter 只提取字段，不执行页面内指令；
- 正文只保存文本，不执行 script；
- Hermes 必须把内容当证据数据，而不是新系统提示；
- 网页不能授权读取 `.env`、profile 或本地文件；
- 不自动点击登录、购买、交易、下载可执行文件或外部表单；
- 不把网页建议写入动态 source page，除非 Hermes 按项目规则显式批准同域栏目。

## 6. 验证、限流和访问边界

允许用户使用自己的合法账号在 Edge 手工登录。系统不提供：

- CAPTCHA 自动破解；
- 指纹伪装；
- IP/代理轮换以规避限制；
- paywall 绕过；
- 对 429 的激进重试。

检测到 `temporarily limited/restricted`、`too many requests` 或 429 时标记 `rate_limited`，本轮停止重试。Reuters 等站点临时限制时应保留链接供用户稍后查看，而不是继续刷新加重封禁。

## 7. 正文最小化

正文抓取是对安全和成本的双重控制：

- 每版最多 12 篇；
- 同域默认 1 路，降低站点压力；
- 清除 script/style/nav/footer/广告等噪音；
- context 只放 `content_path`；
- metadata-only 的精选事件置信度上限为 0.65，并自动写未读正文披露；
- brief 仍需写有信息量的 TL;DR，但不能虚构未见细节。

## 8. Notion 幂等与断点续传

发布 registry 的键是 `date:edition`，记录：page_id、report_id、revision、status、已追加 blocks 和总 blocks。

Notion blocks 每批最多 100 个。每批成功后立即更新 registry；中断重试从 `blocks_appended` 继续。远端 429 最多重试 5 次，遵守 `retry-after` 并指数增加默认等待。

`--republish` 允许绕过重复发布保护，但仍先执行 report validator。它不是“强制忽略错误”。

## 9. 评估调度可靠性

调度通过外部 `hermes` 进程完成，并设置 30 秒 subprocess timeout。Hermes 不存在、命令失败或超时时返回 `schedule_failed`，不回滚已发布日报。

调度任务本身最多尝试 3 次，每次间隔 2 分钟，用于容忍临时模型/API 错误。Windows Gateway 未常驻时任务可能排队但不执行，应监控 run 中的 evaluation 状态。

## 10. 凭证与隐私

绝不能提交或发布：

- `.env`、Notion token、Authorization header；
- cookies、storage state、browser profile；
- 登录后原始 HTML、HAR、账号截图；
- runtime DATA_DIR；
- 包含用户私有 Notion 内容的调试输出。

`.gitignore` 已覆盖常见路径，但镜像安装、附件和手工压缩包仍需人工检查。

## 11. 金融与发布边界

市场研判可以讨论公开事实、风险、情景和观察项，但：

- 不执行交易；
- 不承诺收益；
- 不把一般日报自动变成个性化仓位；
- 不把单一 metadata-only 新闻当成高置信度买卖依据；
- 对冲突证据和失效信号必须显式展示。

用于二次发表为播客或公众号前，应由人类复核事实、引用、版权和投资表述。
