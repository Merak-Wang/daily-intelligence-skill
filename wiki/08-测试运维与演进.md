# 测试与发布

## 本地检查

```powershell
python -m pip install -e ".[dev]"
python -m ruff check .
python -m pytest
python -m compileall -q src tests
daily-intel --help
git diff --check
```

CI 在 Windows 和 Ubuntu 上运行 Python 3.11、3.12，执行 Ruff、pytest 和 compileall。

## 测试分层

| 文件 | 覆盖范围 |
| --- | --- |
| `tests/test_config.py` | 配置、路径、数据根、动态栏目 |
| `tests/test_normalize.py` | URL、标题、日期和来源过滤 |
| `tests/test_prefetch.py` | HTTP 预取和并发限制 |
| `tests/test_importer.py` | 旧 source-index 导入和状态修复 |
| `tests/test_reporting.py` | Schema、编译、证据和语义校验 |
| `tests/test_architecture.py` | Workflow、Artifact、渲染、Notion 和恢复 |
| `tests/test_semantics.py` | 语义缓存和 Evaluation 门槛 |
| `tests/skills/` | Skill 入口和发布续传 |

测试不访问真实 Notion，不使用真实 Token，不依赖已有浏览器 Profile。外部服务通过 Fake Client、Mock Transport 或临时页面替代。

## 变更与测试

| 变更 | 最低测试要求 |
| --- | --- |
| 来源过滤 | 接受、拒绝、去重和日期样例 |
| 状态判断 | 成功、空结果、失败、验证和限流 |
| 并发 | 全局上限、同域上限和异常释放 |
| 报告字段 | Schema、编译、跨字段校验和所有投影 |
| 存储 | 原子写、Revision、锁和跨数据根拒绝 |
| Notion | schema 不匹配、幂等、中断续传和重发 |
| 兼容逻辑 | 至少一个旧格式 Fixture |

## 手动冒烟

自动测试不能覆盖真实网站变化和浏览器交互。发布前按变更范围选择执行：

- `daily-intel collect --edition morning --source <id>`；
- `daily-intel verify-pending --index <path>`；
- 使用合成报告生成 HTML 和 PDF；
- 检查窄屏 HTML、打印分页和链接；
- 使用测试数据库验证 Notion schema 和续传。

不要为了冒烟测试向生产 Notion 数据库发布，也不要提交冒烟产生的 Profile、截图、HTML 或运行数据。

## 故障定位

| 现象 | 首先检查 |
| --- | --- |
| 来源显示 `no_items` | HTTP 状态、挑战分类、URL 过滤 |
| 同一新闻重复 | `canonicalize_url()` 和来源内去重 |
| 正文未进入 Context | 选中 ID、硬上限、Index Revision 和 lineage |
| 报告校验失败 | 错误路径、Index 身份、固定栏目和分析域 |
| Run 无法恢复 | Run 状态、锁文件、数据根和 Artifact 路径 |
| Notion 404 | Connection 授权和 `data_source_id` |
| Notion 字段错误 | `configs/notion.yaml` 与实际 schema |
| Evaluation 未应用 | 报告 ID、Hash、评分门槛和排除范围 |

用户侧恢复命令见 `references/runbook.md`。

## 发布检查

1. 工作区只包含预期改动；
2. 没有 `.env`、运行数据、浏览器文件、认证页面或账户截图；
3. 版本号、`CHANGELOG.md` 和 Release Notes 一致；
4. 合成样例通过当前 Schema；
5. 完整检查通过；
6. README 中的安装命令和相对链接有效；
7. 构建产物位于 `dist/`，不提交仓库；
8. Release 页面记录兼容变化和已知限制。

构建过程的临时验收记录不放在仓库根目录，也不替代 CI 结果。
